# 🔥 Firebase Setup Guide — Fashion Store App
## Step-by-Step Instructions for Complete Beginners

---

## STEP 1 — Create a Firebase Project

1. Open your browser and go to: https://console.firebase.google.com
2. Click **"Add project"**
3. Type a project name: `fashion-store-cit211`
4. Click **Continue**
5. Turn OFF Google Analytics (not needed) → Click **Create project**
6. Wait for it to finish → Click **Continue**

---

## STEP 2 — Add Android App to Firebase

1. On the Firebase project homepage, click the **Android icon** (looks like a robot)
2. Fill in the form:
   - **Android package name:** `com.example.fashion_store`
     *(This must match your app's applicationId in android/app/build.gradle)*
   - **App nickname:** Fashion Store
   - **Debug signing certificate:** Leave blank for now
3. Click **Register app**
4. Click **Download google-services.json**
5. **IMPORTANT:** Copy the downloaded `google-services.json` file into:
   ```
   android/app/google-services.json
   ```
6. Click **Next** → **Next** → **Continue to console**

---

## STEP 3 — Enable Firebase Authentication

1. In the Firebase console left menu, click **Authentication**
2. Click **Get started**
3. Click on **Email/Password**
4. Toggle **Enable** to ON
5. Click **Save**

Done! Users can now register and login.

---

## STEP 4 — Create Firestore Database

1. In the Firebase console left menu, click **Firestore Database**
2. Click **Create database**
3. Select **Start in test mode** (allows all reads/writes for 30 days)
4. Click **Next**
5. Choose a location (pick the one closest to you, e.g., `asia-south1` for Sri Lanka)
6. Click **Enable**

Wait for it to finish setting up.

---

## STEP 5 — Set Firestore Security Rules

1. In Firestore, click the **Rules** tab at the top
2. Replace everything with:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Products: anyone can read, no one can write from app
    match /products/{productId} {
      allow read: if true;
      allow write: if false;
    }

    // Orders: only the user who created it can read/write
    match /orders/{orderId} {
      allow read, write: if request.auth != null
                         && request.auth.uid == resource.data.userId;
      allow create: if request.auth != null;
    }

    // Users: only the user themselves can read/write their profile
    match /users/{userId} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }
  }
}
```

3. Click **Publish**

---

## STEP 6 — Install FlutterFire CLI and Connect Firebase

Open your terminal (Command Prompt or VS Code terminal):

### 6a. Install FlutterFire CLI
```bash
dart pub global activate flutterfire_cli
```

### 6b. Log in to Firebase
```bash
firebase login
```
A browser window will open — log in with the same Google account you used for Firebase.

### 6c. Connect your app to Firebase
Navigate to your project folder first, then run:
```bash
cd path/to/your/fashion_store
flutterfire configure
```

When it asks:
- Select your project: choose `fashion-store-cit211`
- Select platforms: choose **android** (press Space to select, Enter to confirm)

This will automatically create the file `lib/firebase_options.dart` — DO NOT edit this file.

---

## STEP 7 — Update pubspec.yaml

Replace your `pubspec.yaml` dependencies with the one provided (`pubspec.yaml` file in this package).

Then run:
```bash
flutter pub get
```

---

## STEP 8 — Update android/build.gradle

Open `android/build.gradle` and make sure it has:
```gradle
buildscript {
    dependencies {
        classpath 'com.google.gms:google-services:4.4.0'
    }
}
```

---

## STEP 9 — Update android/app/build.gradle

Open `android/app/build.gradle` and add at the very bottom:
```gradle
apply plugin: 'com.google.gms.google-services'
```

Also make sure `minSdkVersion` is at least **21**:
```gradle
android {
    defaultConfig {
        minSdkVersion 21
    }
}
```

---

## STEP 10 — Add Sample Products to Firestore

Your Firestore starts empty. You need to add products.

**Option A — Use the seed function (easiest)**

In your `HomeScreen` or `main.dart`, temporarily add this call ONCE:

```dart
// Add this temporarily in initState() of HomeScreen
import '../services/product_service.dart';

@override
void initState() {
  super.initState();
  // UNCOMMENT THE LINE BELOW, run the app ONCE, then comment it back out
  // ProductService().seedProducts();
}
```

Run the app once, then REMOVE that line. Your Firestore will now have 6 products.

**Option B — Add manually in Firebase Console**

1. Go to Firestore → click **Start collection**
2. Collection ID: `products`
3. Add a document with these fields:
   - `name` (string): Classic White Tee
   - `brand` (string): Urban Basics
   - `price` (number): 29.99
   - `rating` (number): 4.5
   - `reviewCount` (number): 128
   - `category` (string): Men
   - `isFeatured` (boolean): true
   - `imageUrl` (string): https://images.unsplash.com/photo-1521572163474-6864f9cf17ab?w=400
   - `sizes` (array): S, M, L, XL
   - `colors` (array): White, Black, Gray
   - `description` (string): Premium cotton tee

## TROUBLESHOOTING

| Problem | Fix |
|---|---|
| `google-services.json not found` | Make sure file is in `android/app/` not `android/` |
| `firebase_options.dart not found` | Run `flutterfire configure` again |
| `minSdkVersion` error | Set `minSdkVersion 21` in `android/app/build.gradle` |
| Products not showing | Run the `seedProducts()` method once |
| `PERMISSION_DENIED` from Firestore | Check your Firestore security rules in Step 5 |
| Build fails after adding Firebase | Run `flutter clean` then `flutter pub get` |
